# Impeccable Design SOP (OpenClaw x Lulu)

## Goal
Use Impeccable design skills to convert raw content into audience-fit frontend style, then deliver implementation-ready UI in GitHub for review.

## Command Set (core)
- critique, adapt, typeset, colorize, animate
- polish, audit, optimize
- bolder / quieter (tension control)
- overdrive (optional, high-stylization)

## Conversation Contract (with user)
Before coding, always confirm:
1) Audience (who reads this?)
2) Outcome (what should they feel/do?)
3) Brand tone (calm/professional/energetic/editorial/etc.)
4) Risk tolerance (safe / experimental)

Then propose 2 style routes:
- Route A (conservative)
- Route B (expressive)

User chooses one route before implementation.

## Execution Flow
1. /critique content mood + rhetorical structure
2. /adapt to selected style route
3. /typeset for hierarchy + reading rhythm
4. /colorize palette from tone and audience
5. /animate with restraint (or /overdrive if requested)
6. /polish spacing/contrast/details
7. /audit (report-only quality gate)
8. Implement frontend and push to GitHub

## Output Requirements
Each delivery must include:
- Style rationale (why this style fits this audience)
- Typography choices
- Color palette (hex + usage)
- Motion rules
- Accessibility notes (contrast/readability)
- Preview checklist

## Acceptance Checklist
- Visual consistency
- Readability across devices
- Interaction clarity
- Matches requested tone
- GitHub commit link provided

## Default Rule
No generic AI-looking UI.
Always style by audience + content intent.
